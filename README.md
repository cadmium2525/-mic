# マリオクラフト (Mario × Minecraft Fan Game)

横スクロールアクション × 採掘 × クラフトを組み合わせた同人ゲームです。
ビルド不要の純粋な HTML/CSS/JavaScript (Canvas 2D) で作られており、
**そのまま GitHub Pages で公開できます**。

現在は **ワールド1 (1-1 〜 1-4)** を収録しています。エンジンとレベル定義は
分離されており、`js/levels.js` に新しいステージ定義を追加するだけで
ワールド2以降を拡張できる設計になっています。

## 遊び方

スマホ・タブレットでは画面上の仮想パッド（左下: 移動、右下: JUMP/採掘/設置）で
操作できます。PC ではキーボードで操作します。

| キー / タッチ | アクション |
|---|---|
| ← → / 左下パッド | 移動 |
| Space・↑ / JUMPボタン | ジャンプ |
| X / 採掘ボタン | 採掘 / 敵への攻撃（剣所持時） |
| ↓ + X / 下+採掘ボタン | 足元のブロックを採掘 |
| Z / 設置ボタン | 選択中のアイテムを設置 |
| 1 / 2 / 3 / ホットバータップ | 設置アイテム(ラダー/たいまつ/足場)を選択 |
| C / ⚒ボタン | クラフトメニューを開閉 |
| Esc・P / ❚❚ボタン | ポーズ |

画面が縦向きのスマホでアクセスすると、横向き推奨のバナーが表示されます
（閉じると再表示されません）。

木・葉・草・土は素手で採掘できますが、石や鉱石は対応するツルハシが
必要です（Minecraft のティア制を踏襲）。

### 装備の切り替え（素手 / ツルハシ / 剣）

採掘ボタン(X)の挙動は「今何を手に持っているか」で変わります。ホットバー
左側の3スロット（✋素手 / ⛏ツルハシ / ⚔剣）をタップ、またはキーボードの
**Q** で切り替えます。ツルハシや剣を初めてクラフトすると自動的にそれを
手に持った状態になります。

- **✋素手**：木・葉・草・土など requiredTool:none のブロックのみ採掘可
- **⛏ツルハシ**：所持している最高ランクのツルハシで石・鉱石を採掘（敵への攻撃はできません）
- **⚔剣**：正面の敵を攻撃（ブロックは採掘できません）。踏みつけはどの装備でも可能

### クラフトレシピ（抜粋）

- 木のツルハシ：木材×2 + 枝×1 → 石・石炭が採掘可能に
- 石のツルハシ：石×3 + 枝×1 → 鉄鉱石が採掘可能に
- 鉄のツルハシ：鉄×3 + 枝×1 → ダイヤモンドが採掘可能に
- 木の剣／ダイヤの剣：敵を正面から撃破（ダイヤの剣はボスを一撃）
- ラダー／たいまつ：設置してよじ登る・暗闇を照らす
- 足場ブロック：石×2 **または** 土×3 → 穴や溶岩をふさげる置きブロックが3個手に入る
  （草・土ブロックを掘ると誰でも簡単に手に入る「土」を使った安価なレシピと、
  やや頑丈な「石」のレシピの2種類があります）

## iPhone で快適に遊ぶ（PWA / ホーム画面への追加）

Safari のアドレスバー分だけ画面が狭くなるのを避けるため、本作は
**PWA（ホーム画面に追加すると独立したアプリのように全画面で起動する）**
に対応しています。iPhone 12 を基準に、ホーム画面追加後の全画面表示・
横向き表示・ノッチ／ホームインジケーターのセーフエリアを考慮したレイアウト
になっています。

1. Safari で公開先の URL を開く
2. 共有ボタン（□に↑）をタップ → **「ホーム画面に追加」** を選択
3. ホーム画面のアイコンから起動するとアドレスバーなしの全画面で遊べます
4. 本体を横向きにすると自動でコンパクトな横画面レイアウトに切り替わります

オフラインキャッシュ用の Service Worker (`sw.js`) も同梱しているため、
一度読み込めば電波が無い状態でも起動できます。キャッシュを更新したい
場合は `sw.js` 内の `CACHE_NAME` の値（例: `mariocraft-v1` → `v2`）を
変更してください。

## ローカルで試す

ブラウザは `file://` から ES Modules 等を読めない場合があるため、
簡易サーバー経由での起動を推奨します。

```bash
cd mariocraft
python3 -m http.server 8000
# http://localhost:8000 を開く
```

（本プロジェクトは `<script>` タグの読み込みのみで構成されているため、
実際には `file://` で `index.html` を直接開いても動作します）

## GitHub Pages への公開手順

1. このフォルダの中身（`index.html` / `style.css` / `js/`）をリポジトリの
   ルート（またはお好みのフォルダ）にそのままコミット＆プッシュします。
2. GitHub リポジトリの **Settings → Pages** を開きます。
3. **Source** を `Deploy from a branch` にし、公開したいブランチ
   （例: `main`）とフォルダ（`/root` または `index.html` を置いたフォルダ）
   を選択して **Save** します。
4. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` でゲームが
   公開されます。

外部依存は Google Fonts（ドット絵フォント "Press Start 2P"）の CDN 読み込み
のみで、画像素材は一切使用していません（すべて Canvas 上でコードから
ドット絵を描画しています）。

## ファイル構成

```
index.html          画面レイアウト・UIオーバーレイ
style.css            ドット絵風スタイル
manifest.json         PWAマニフェスト(ホーム画面追加・全画面表示用)
sw.js                  Service Worker(オフラインキャッシュ)
icons/                 PWA用アイコン(コードで生成したドット絵)
js/sprites.js        8x8ドットグリッドの描画エンジン（自作アセット）
js/tiles.js           タイル種別・採掘可否ルール
js/levels.js          ステージ生成システム + ワールド1(1-1〜1-4)の定義
js/crafting.js        素材・レシピ・クラフトUI
js/entities.js         敵(ウォーカー/スライム/ボス)のAI
js/player.js           プレイヤーの物理・採掘・インベントリ・装備切替
js/engine.js            メインループ・カメラ・当たり判定・描画
js/main.js               画面状態管理(タイトル/プレイ/ポーズ/クラフト/クリア)
```

## 今後の拡張について（ワールド2〜8）

`js/levels.js` の `Game.LevelDefs` 配列に新しいステージ定義オブジェクトを
追加し、`Game.Levels` に含めるだけで新ワールドを追加できます。敵タイプや
鉱石、クラフトレシピを増やしたい場合は `js/entities.js` / `js/tiles.js` /
`js/crafting.js` にそれぞれ定義を足してください。
