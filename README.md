# Craft & Jump - World1（スーパーマリオ x Minecraft 横スクロールアクション）

ビルド不要・外部ライブラリ不要のHTML5(Canvas)横スクロールアクションゲームです。
GitHub Pagesにそのまま配置して遊べます。

## 遊び方（ローカルで試す）

ES Modules と `fetch` を使っているため、`index.html` を直接ダブルクリックしては動きません。
必ず簡易サーバー経由で開いてください。

```bash
cd game
python3 -m http.server 8080
# ブラウザで http://localhost:8080/ を開く
```

## 操作方法

- 左下：左右移動
- 右下（小さいボタン）：採掘／設置／会話（状況に応じて自動的に切り替わります）
- 右下（大きいボタン）：ジャンプ
- 右上：🎒インベントリ／🛠️クラフト
- PCではキーボードでも操作できます（矢印キー or WASD, Space=ジャンプ, E/F=アクション）

## ディレクトリ構成

```
game/
├── index.html          # エントリーポイント（Canvas + スマホ用UI）
├── css/style.css        # 見た目・レイアウト
├── js/
│   ├── main.js          # 起動処理
│   ├── core/            # ゲームループ・入力・カメラ・物理・セーブ・アセット生成
│   ├── entities/        # プレイヤー・敵・アイテム・商人
│   ├── stage/           # タイルマップ・ステージ読み込み
│   ├── systems/         # インベントリ・クラフト・会話
│   └── ui/               # DOM UI（パネル・HUD・オーバーレイ）
└── data/
    ├── items.json       # 素材・道具の定義
    ├── recipes.json     # クラフトレシピ
    ├── enemies.json     # 敵のパラメータ
    ├── merchants.json   # 商人の販売プール
    ├── dialogues.json   # 会話イベント
    └── stages/1-1.json ... 1-4.json  # 各ステージのタイルマップ・配置データ
```

## 設計方針

- **ロジックと描画の分離**：`Game._update()` が状態更新、`Game._render()` が描画のみを担当。
- **データ駆動**：ステージ地形・敵配置・アイテム・レシピ・商人の品揃え・会話はすべてJSON。
  コードを変更せずJSON追加だけで新しいステージ／ワールドを増やせる構成です。
- **ドット絵は自作**：外部画像ファイルを一切使わず、`js/core/AssetFactory.js` 内の文字パターンから
  Canvasに焼き込んで生成しています（`buildSprite`）。

## 新しいステージ／ワールドの追加方法

1. `data/stages/2-1.json` のような新しいステージJSONを作成する
   （`tiles`, `legend`, `enemies`, `items`, `playerStart`, `goal` などを定義）。
2. `js/core/DataLoader.js` の `STAGE_LIST` にエントリを追加する。
3. 必要なら `data/enemies.json` / `data/merchants.json` / `data/dialogues.json` に新しい定義を追記する。

コード側の変更は基本的に不要です。

## 現在の実装状況（ワールド1・全4ステージ）

| ステージ | テーマ | 内容 |
|---|---|---|
| 1-1 | 草原 | 基本操作（移動・ジャンプ・敵踏みつけ）、商人との取引 |
| 1-2 | 地下鉱山 | 暗闇の中での素材採掘、松明クラフトのチュートリアル |
| 1-3 | 森 | 石の足場をクラフトして谷を渡るブロック設置ギミック |
| 1-4 | 城 | 鍵の欠片集め→鍵をクラフトして扉を開け、ボス（石のゴーレム）を倒すクラフト必須ボス戦 |

セーブデータは `localStorage` に保存され、所持素材・コイン・入手済み道具・現在のステージが
次回起動時に復元されます。
